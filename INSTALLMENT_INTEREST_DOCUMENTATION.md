# مستندات سیستم سود ماهانه برای خریدهای اقساطی

## توضیحات

این سیستم به شما امکان می‌دهد که برای خریدهای اقساطی، هر ماه یک درصد سود به مبلغ مانده تعلق بگیرد.

## نحوه کار

1. **نرخ سود ماهانه** از ستینگ `installment_monthly_interest_rate` خوانده می‌شود (به صورت درصد، مثلاً 2 برای 2%)
2. برای هر ماه، سود بر اساس **مبلغ مانده** محاسبه می‌شود
3. مجموع سودها به مبلغ اصلی اضافه می‌شود
4. مبلغ کل (با سود) به تعداد اقساط تقسیم می‌شود

## مثال محاسبه

فرض کنید:
- مبلغ خرید: 1,000,000 تومان
- تعداد اقساط: 4 ماه
- نرخ سود ماهانه: 2%

محاسبه:
- **ماه 1**: مانده = 1,000,000، سود = 1,000,000 × 0.02 = 20,000
- **ماه 2**: مانده = 750,000 (بعد از پرداخت قسط اول)، سود = 750,000 × 0.02 = 15,000
- **ماه 3**: مانده = 500,000، سود = 500,000 × 0.02 = 10,000
- **ماه 4**: مانده = 250,000، سود = 250,000 × 0.02 = 5,000

**مجموع سود**: 20,000 + 15,000 + 10,000 + 5,000 = 50,000 تومان
**مبلغ کل**: 1,000,000 + 50,000 = 1,050,000 تومان
**مبلغ هر قسط**: 1,050,000 ÷ 4 = 262,500 تومان

## تنظیمات

### افزودن ستینگ نرخ سود

فایل SQL را اجرا کنید:
```sql
-- فایل: database/migrations/add_installment_interest_rate_setting.sql
INSERT INTO `settings` (`key`, `value`, `created_at`, `updated_at`) 
VALUES ('installment_monthly_interest_rate', '2', NOW(), NOW())
ON DUPLICATE KEY UPDATE `value` = '2', `updated_at` = NOW();
```

### تغییر نرخ سود

از طریق API:
```http
PUT /api/settings/installment_monthly_interest_rate
Content-Type: application/json

{
  "value": "2"
}
```

یا مستقیماً در دیتابیس:
```sql
UPDATE `settings` 
SET `value` = '2' 
WHERE `key` = 'installment_monthly_interest_rate';
```

### غیرفعال کردن سود

برای غیرفعال کردن سود، مقدار را `0` قرار دهید:
```sql
UPDATE `settings` 
SET `value` = '0' 
WHERE `key` = 'installment_monthly_interest_rate';
```

## نکات مهم

1. اگر نرخ سود `0` باشد یا ستینگ وجود نداشته باشد، سود محاسبه نمی‌شود
2. سود بر اساس **مبلغ مانده** در هر ماه محاسبه می‌شود (نه مبلغ کل)
3. مبلغ `total_amount` در جدول `purchases` شامل سود می‌شود
4. مبلغ `installment_amount` در جدول `purchases` مبلغ هر قسط (با سود) است
